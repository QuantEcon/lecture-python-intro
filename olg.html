20. The Overlapping Generations Model#

In this lecture we study the overlapping generations (OLG) model.

The dynamics of this model are quite similar to Solow-Swan growth model.

At the same time, the OLG model adds an important new feature: the choice of how much to save is endogenous.

To see why this is important, suppose, for example, that we are interested in predicting the effect of a new tax on long-run growth.

We could add a tax to the Solow-Swan model and look at the change in the steady state.

But this ignores something important: households will change their behavior when they face the new tax rate.

Some might decide to save less, and some might decide to save more.

Such changes can substantially alter the predictions of the model.

Hence, if we care about accurate predictions, we should model the decision problems of the agents.

In particular, households in the model should decide how much to save and how much to consume, given the environment that they face (technology, taxes, prices, etc.)

The OLG model takes up this challenge.

We will present a simple version of the OLG model that clarifies the decision problem of households and studies the implications for long run growth.

20.2. Supply of capital#

20.2.1. Consumer’s problem#

Suppose that utility for individuals born at time \(t\) take the form

(20.1)#\[ U_t = u(c^1_t) + \beta u(c^2_{t+1})\]

Here

  • \(u: \mathbb R_+ \to \mathbb R\) is the flow utility function satisfying some properties

  • \(\beta \in (0, 1)\) is the discount factor

  • \(c^1_t\) is time \(t\) consumption of the individual born at time \(t\)

  • \(c^2_{t+1}\) is time \(t+1\) consumption of the same individual (born at time \(t\))

Savings by an individual of generation \(t\), \(s_t\), is determined as a solution to:

(20.2)#\[\begin{split} \begin{aligned} \max_{c^1_t, c^2_{t+1}, s_t} \ & \left \{ u(c^1_t) + \beta u(c^2_{t+1}) \right \} \\ \mbox{subject to } \ & c^1_t + s_t \le w_t \\ & c^2_{t+1} \le R_{t+1}s_t\\ \end{aligned}\end{split}\]

where

  • \(w_t\) is the wage rate at time

  • \(R_{t+1}\) is time \(t+1\) rental rate of capital.

The second constraint incorporates the notion that individuals only spend money on their own end of life consumption. Also, Since \(u\) is strictly increasing, both constraints will hold as equalities.

Substituting \(s_t\) from the first constraint into the second we get \(c^2_{t+1}\) in terms of \(c^1_t\), i.e.,

(20.3)#\[ c^2_{t+1} = R_{t+1}(w_t - c^1_t)\]

Thus first-order condition for a maximum can be written in the familiar form of the consumption Euler equation by plugging \(c^2_{t+1}\) into the objective function and taking derivative with respect to \(c^1_t\)

(20.4)#\[ u'(c^1_t) = \beta R_{t+1} u'( R_{t+1} (w_t - c^1_t))\]

From the first constraint we get

(20.5)#\[ c^1_{t} = w_t - s_t\]

With it the Euler equation (4) becomes

(20.6)#\[ u'(w_t - s_t) = \beta R_{t+1} u'( R_{t+1} s_t)\]

From this we can solve for savings analytically or, if necessary, numerically

(20.7)#\[ s_t = s(w_t, R_{t+1})\]

Let \(L_t\) be the time \(t\) labor.

Furthermore let’s assume a constant population size, i.e., \(L_{t+1}=L_t=L\) for all \(t\).

Total savings in the economy will be equal to

(20.8)#\[ S_t = s_t L\]

In our closed economy, net saving this period will be equal to supply next period, i.e.,

(20.9)#\[ K_{t+1} = K^S(w_t, R_{t+1}) = S_t = L s_t = L s(w_t, R_{t+1})\]

Here \(K^S\) is a time-invariant function mapping wage \(w_t\) and capital return rate \(R_{t+1}\) to aggregate capital supply \(K_{t+1}\).

20.2.2. Special case: log preference#

Assume \(u(c) = \log c\).

The Euler equation simplifies to

(20.10)#\[ s_t= \beta (w_t - s_t)\]

Solving for saving,

(20.11)#\[ s_t = s(w_t, R_{t+1}) = \frac{\beta}{1+\beta} w_t\]

And hence

(20.12)#\[ K_{t+1} = K^s(R_{t+1}) = Ls_t = L \frac{\beta}{1+\beta} w_t\]

20.3. Demand for capital#

20.3.1. Firm’s problem#

For each integer \(t \geq 0\), output \(Y_t\) in period \(t\) is given by

(20.13)#\[ Y_t = K_t^{\alpha} L_t^{1-\alpha}\]

Here \(K_t\) is capital, \(L_t\) is labor, and \(\alpha\) is the output elasticity of capital in the Cobb-Douglas production function.

Demand for labor \(L\) and capital \(K_t\) is determined by the profit maximization problem

(20.14)#\[ \max_{K_t, L} \{ K^{\alpha}_t L^{1-\alpha} - R_t K_t - L w_t \}\]

20.3.2. Demand for capital#

The first-order conditions for a maximum can be obtained by taking the derivative of the objective function with respect to capital and labor respectively and setting it to zero:

(20.15)#\[ (1-\alpha)(K_t / L)^{\alpha} = w_t\]

and

(20.16)#\[ \alpha (K_t / L)^{\alpha - 1} = R_t\]

Rearranging (20.16) gives the aggregate demand for capital

(20.17)#\[ K_{t+1} = K^d (R_{t+1}) = L \left (\frac{R_{t+1}}{\alpha} \right )^{1/(\alpha - 1)}\]

20.4. Equilibrium#

The equality of aggregate demand (20.17) and aggregate supply (20.12) for capital yields the equalibrium.

Specifically we have

(20.18)#\[ K^s(R_{t+1}) = K^d(R_{t+1})\]

or, equivalently,

(20.19)#\[ L\frac{\beta}{1+\beta} (1-\alpha)(K_t / L)^{\alpha} = L\left (\frac{R_{t+1}}{\alpha} \right )^{1/(\alpha - 1)}\]

Then we can solve for the equilibrium price

(20.20)#\[ R^*_{t+1} = \alpha \left ( \frac{\beta (1-\alpha)(K_t / L)^{\alpha}}{1+\beta} \right )^{\alpha - 1}\]

Plugging it into either the demand or the supply function gives the equilibrium quantity

(20.21)#\[ K^*_{t+1} = \frac{\beta }{1+\beta} (1-\alpha)(K_t / L)^{\alpha} L\]
Model = namedtuple('Model', ['α',        # output elasticity of capital in the Cobb-Douglas production function
                             'β',        # discount factor
                             'u',        # parameter which defines the flow utility
                             'L',        # population size
                             'u_params'] # other params used to define u
                   )

def u(c):
    return np.log(c)

def create_olg_model(α=0.3, β=0.9, u=u, L=10.0, u_params=dict()):
    return Model(α=α, β=β, u=u, L=L, u_params=u_params)

def equilibrium(model, K_prev):
    α, β, L = model.α, model.β, model.L
    R = α * ( β * (1-α) * (K_prev / L)**α / (1+β))**(α-1)
    K = β / (1+β) * (1-α) * (K_prev / L)**α * L
    return R, K

def aggregate_capital_demand(R, model, K_prev):
    α, L = model.α, model.L
    return (R/α)**(1/(α-1)) * L

def aggregate_capital_supply(R, model, K_prev):
    α, β, L = model.α, model.β, model.L
    λ = np.ones_like(R)
    return λ * β / (1+β) * (1-α) * (K_prev / L)**α * L

20.5. Dynamics and steady state#

Let \(k_t := K_t / L\).

Aggregate supply of capital (20.12) becomes

(20.22)#\[ k_{t+1} = k^s(R_{t+1}) = \frac{\beta}{1+\beta} w_t\]

(20.15) becomes

(20.23)#\[ (1-\alpha)(k_t)^{\alpha} = w_t\]

Combining (20.22) and (20.23) yields the law of motion for capital

(20.24)#\[ k_{t+1} = \frac{\beta}{1+\beta} (1-\alpha)(k_t)^{\alpha}\]

A steady state can be solved analytically in this case.

That is, \(k_{t+1} = k_t = k^*\), where

(20.25)#\[ k^* = \frac{\beta (1-\alpha) (k^*)^{\alpha}}{(1+\beta)}\]

We can solve this equation to obtain

(20.26)#\[ k^* = \left (\frac{\beta (1-\alpha)}{1+\beta} \right )^{1/(1-\alpha)}\]
def k_update(k, model):
    α, β = model.α, model.β
    return β * (1 - α) * k**α /  (1 + β)

def plot_45(m, k_update, kstar=None):
    kmin, kmax = 0, 0.6
    x = 1000
    k_grid = np.linspace(kmin, kmax, x)
    k_grid_next = np.empty_like(k_grid)

20.6. Another special case: CRRA preference#

Let’s now assume that the model is the same except that \(u(c) = \frac{ c^{1- \gamma}-1}{1-\gamma}\), where \(\gamma >0, \gamma\neq 1\).

def crra(c, γ=0.5):
    return c**(1 - γ) / (1 - γ)

m_crra = create_olg_model(u=crra, u_params={'γ': 0.5})

20.6.1. New aggregate supply#

For households, the Euler equation becomes

(20.27)#\[ (w_t - s_t)^{-\gamma} = \beta R^{1-\gamma}_{t+1} (s_t)^{-\gamma}\]

Solving for savings, we have

(20.28)#\[ s_t = s(w_t, R_{t+1}) = w_t \left [ 1 + \beta^{-1/\gamma} R_{t+1}^{(\gamma-1)/\gamma} \right ]^{-1}\]

With the CRRA utility, the aggregate supply of capital (20.9) becomes

(20.29)#\[ K_{t+1} = K^S(w_t, R_{t+1}) = S_t = L s_t = L s(w_t, R_{t+1}) = L w_t \left [ 1 + \beta^{-1/\gamma} R_{t+1}^{(\gamma-1)/\gamma} \right ]^{-1}\]

20.6.2. Equilibrium#

The equality of aggregate demand (20.17) and new aggregate supply (20.29) for capital yields the equalibrium.

Specifically we have

(20.30)#\[ K^S(R_{t+1}) = K^d(R_{t+1})\]

or equivalently

(20.31)#\[ L w_t \left [ 1 + \beta^{-1/\gamma} R_{t+1}^{(\gamma-1)/\gamma} \right ]^{-1} = L\left (\frac{R_{t+1}}{\alpha} \right )^{1/(\alpha - 1)}\]

Combining with (20.15) yields

(20.32)#\[ L (1-\alpha)(K_t / L)^{\alpha} \left [ 1 + \beta^{-1/\gamma} R_{t+1}^{(\gamma-1)/\gamma} \right ]^{-1} = L\left (\frac{R_{t+1}}{\alpha} \right )^{1/(\alpha - 1)}\]

Here we cannot solve for the equilibrium price and quantity analytically.

20.6.3. Dynamics and steady state#

Setting \(k_t := K_t / L\) and using (20.28).

Aggregate supply of capital (20.9) becomes

(20.33)#\[ k_{t+1} = k^s(R_{t+1}) = \left [ 1 + \beta^{-1/\gamma} R_{t+1}^{(\gamma-1)/\gamma} \right ]^{-1} w_t\]

(20.16) becomes

(20.34)#\[ \alpha k^{\alpha - 1}_t = R_t\]

Combing with (20.23) and (20.33) gives

(20.35)#\[ k_{t+1} = \left [ 1 + \beta^{-1/\gamma} (\alpha k^{\alpha - 1}_{t+1})^{(\gamma-1)/\gamma} \right ]^{-1} (1-\alpha)(k_t)^{\alpha}\]

Note that with this equation and \(k_t\) as given we cannot solve for \(k_{t+1}\) by pencil and paper.

That is we don’t have an analytical solution for the sample path \(\{k_{t+1}\}\) now.

To solve for \(k_{t+1}\) we need to turn to newton’s method.

Suppose,

(20.36)#\[ f(k_{t+1}; k_t)=k_{t+1} \left [ 1 + \beta^{-1/\gamma} \left ( \alpha k^{\alpha-1}_{t+1} \right )^{(\gamma-1)/\gamma} \right ] - (1-\alpha) k^{\alpha}_t =0\]

If \(k_t\) is given then \(f(\cdot)\) is a function of unknown \(k_{t+1}\).

Then we can use scipy.optimize.newton to solve the Equation \(f(k_{t+1})=0\) for \(k_{t+1}\).

First let define \(f(\cdot)\).

def f(k_prime, k, model):
    α, β, γ = model.α, model.β, model.u_params['γ']
    z = (1 - α) * k**α
    R1 = α ** (1-1/γ)
    R2 = k_prime**((α * γ - α + 1) / γ)
    p = k_prime + k_prime * β**(-1/γ) * R1 * R2
    return p - z

Let’s define a function k_next that finds the value of \(k_{t+1}\).

def k_next(k_prime, model):
    return optimize.newton(f, k_prime, args=(k_prime, model))

plot_45(m_crra, k_next, kstar=None)

Unlike the log preference case now a steady state cannot be solved analytically.

To see this recall that, a steady state can be obtained by setting (20.35) to \(k_{t+1} = k_t = k^*\), i.e.,

(20.37)#\[ k^* = \frac{(1-\alpha)(k^*)^{\alpha}}{ 1 + \beta^{-1/\gamma} (\alpha (k^*)^{\alpha-1})^{(\gamma-1)/\gamma}}\]

Similarly we can solve for \(k^*\) using newton’s method.

Suppose that

(20.38)#\[ g(k^*) = k^* \left [ 1 + \beta^{-1/\gamma} (\alpha (k^*)^{\alpha-1})^{(\gamma-1)/\gamma} \right ] - (1-\alpha)(k^*)^{\alpha}\]
def g(k_star, model):
    α, β, γ = model.α, model.β, model.u_params['γ']
    z = (1 - α) * k_star**α
    R1 = α ** (1-1/γ)
    R2 = k_star**((α * γ - α + 1) / γ)
    p = k_star + k_star * β**(-1/γ) * R1 * R2
    return p - z

k_star = optimize.newton(g, 0.2, args=(m_crra,))
print(f"k_star = {k_star}")

k_star = 0.31408046100809445

plot_45(m_crra, k_next, k_star)

Solution to Exercise 20.1

To solve the equation we need to turn to Newton’s method.

The function find_Rstar is used to find \(R^*_{t+1}\) by finding the zero of equation (20.32) using the helper function find_Rstar_newton for a given value of \(K_t\).

Similary, find_Kstar finds the equilibrium quantity \(K^*_{t+1}\) using the value of \(R^*_{t+1}\).

def find_Rstar_newton(x, K_prev, model):
    α, β, γ, L = model.α, model.β, model.u_params['γ'], model.L
    lhs = L * (1-α) * (K_prev / L)**α
    lhs /= (1 + β**(-1/γ) * x**((γ-1)/γ))
    rhs = L * (x / α)**(1/(α-1))
    return lhs - rhs

def find_Rstar(K_prev, model):
    return optimize.newton(find_Rstar_newton, 0.5, args=(K_prev, model))

def find_Kstar(R_star, model):
    return model.L * (R_star / model.α)**(1/(model.α-1))

Solution to Exercise 20.2

To get the time path capital \(\{k_t\}\) first we need to solve the household’s utility maximization problem for the optimal consumption and optimal saving.

With the quasilinear preference the Euler equation (20.6) becomes

(20.39)#\[ 1 + \theta (w_t - s_t)^{\theta-1} = \beta R_{t+1} + \beta R^{\theta}_{t+1} \theta s_t^{\theta - 1}\]

Let \(k_t := K_t / L\).

Since (20.12), (20.23) and (20.34) the Euler equation becomes

(20.40)#\[ 1 + \theta ((1-\alpha)k^{\alpha}_t - k_{t+1})^{\theta-1} = \beta \alpha k^{\alpha - 1}_{t+1} + \beta \theta \alpha^{\theta} k^{\alpha \theta - 1}_{t+1}\]

Obviously \(k_{t+1}\) cannot be solved by pencil and paper.

To solve for \(k_{t+1}\) we need to turn to the newton’s method.

Let’s start by defining the utility function.

def u_quasilinear(c, θ=3):
    return c + c**θ

The function find_k_next is used to find \(k_{t+1}\) by finding the root of equation (20.40) using the helper function solve_for_k_next for a given value of \(k_t\).

def solve_for_k_next(x, k_t, model):
    α, β, L, θ = model.α, model.β, model.L, model.u_params['θ']
    l = 1 + θ * ((1 - α) * k_t**α - x)**(θ - 1)
    r = β * α * x**(α - 1)
    r += β * θ * α**θ * x**(α * θ - 1)
    return l - r

def find_k_next(k_t, model):
    x_vals = np.linspace(0.1, k_t, 200_000)
    k_vals = solve_for_k_next(x_vals, k_t, model)
    return x_vals[np.argmin(np.abs(k_vals-0))]

def find_k_star_q(model):
    x_vals = np.linspace(0.1, 3, 200_000)
    k_vals = solve_for_k_next(x_vals, x_vals, model)
    return x_vals[np.argmin(np.abs(k_vals-0))]

def simulate_ts(k0_values, model, ts_length=6):
    k_star = find_k_star_q(model)

    print("k_star:", k_star)
    fig, ax = plt.subplots(figsize=(10, 5))

    ts = np.zeros(ts_length)

    # simulate and plot time series
    for x_init in k0_values:
        ts[0] = x_init
        for t in range(1, ts_length):
            ts[t] = find_k_next(ts[t-1], model)
        ax.plot(np.arange(ts_length), ts, '-o', ms=4, alpha=0.6,
                label=r'$k_0=%g$' %x_init)
    ax.plot(np.arange(ts_length), np.full(ts_length, k_star),
            alpha=0.6, linestyle='dashed', color='black', label=r'$k^*$')
    ax.legend(fontsize=10)

    ax.set_xlabel(r'$t$', fontsize=14)
    ax.set_ylabel(r'$k_t$', fontsize=14)

    plt.show()
